📖 晋宁会馆·秃贝五边形 4.1 — 项目运维手册
一、项目总览
1.1 项目定位
秃贝五边形是为《罗小黑战记》粉丝社群"晋宁会馆"量身打造的QQ机器人系统。它不是一个通用机器人框架，而是一套深度定制的社群数字化生态，融合了AI对话、RPG修行、社群管理三大核心功能。

1.3 数据存储
text

data/
├── members_db.json    成员档案（QQ号→个人信息）
├── spirit_db.json     灵力数据（QQ号→修行数值）

数据安全机制：

启动时一次性加载到内存，运行时零磁盘IO
每30秒检查脏标记，有变更才写盘
原子写入：先写 .tmp，再 rename 覆盖
每次写盘前自动备份 .bak
关机时强制落盘
二、群分级体系
等级	代号	说明	可用功能
Tier 0	core	核心群（会馆主群、管理群、调试群）	全部功能
Tier 1	allied	联盟群（友好社群）	大部分功能，无馆内专属
Tier 2	public	公开群（任何其他群）	厨房、鉴定、真心话、AI聊天
Tier 3	danger	危险群（特殊限制群）	仅今日老婆+退出彩蛋
配置位置：config/groups.yaml

YAML

core_groups:
  564234162:
    name: "晋宁会馆主群"
    type: "main"        # main=主群 admin=管理群 debug=调试群
  210383914:
    name: "管理组官群"
    type: "admin"

allied_groups:
  1092658209:
    name: "妖灵会馆-顺湖会馆"

danger_groups:
  627896850:
    name: "某群"
添加新群：在对应等级下添加条目，然后执行 /重载配置 即可热生效。

八、参数调整指南
8.1 修改数值参数
文件：config/game_balance.yaml

生效方式：修改后在群内发送 /重载配置，无需重启

常用调参场景：

YAML

# 觉得聚灵收益太低/太高
meditation:
  base_min: 10      # 基础最小值
  base_max: 30      # 基础最大值
  daily_limit: 1    # 每日次数限制

# 觉得厨房太难/太简单
kitchen:
  success_rate: 0.30    # 成功率(0.0-1.0)
  reward_sp: 50         # 成功奖励
  penalty_sp: 10        # 失败惩罚
  daily_limit: 4        # 每日限制

# 觉得嘿咻太频繁/太稀少
heixiu:
  spawn_interval_hours: 30   # 间隔小时数
  spawn_jitter: 21600        # 波动范围(秒)
  catch_success_rate: 0.80   # 捕捉成功率

# 调整等级门槛
levels:
  thresholds:
    2: 100    # Lv.2 需要 100 灵力
    3: 800    # 以此类推

十、故障排查速查表
现象	可能原因	解决方案
秃贝完全无反应	NapCat 断连	docker restart napcat
指令有反应但AI不回	API Key失效/余额不足	检查 .env.prod 中的 SILICONFLOW_API_KEY
数据丢失	异常关机未落盘	从 .bak 文件恢复
嘿咻不刷新	馆禁时间/已有活跃嘿咻	检查时间/重启清除状态
某指令报错	代码bug	查看 journalctl -u jinningbot -f
群里疯狂刷消息	有人刷屏	自动封禁10分钟，或 /封印 QQ 60
配置改了没生效	未重载	/重载配置 或重启服务
